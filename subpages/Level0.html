C# ist Case-Sensitive

Wenn Klassen/Variablen/Funktionen in C# benannt werden, ist die Groß- und Kleinschreibung wichtig. C# ist also Case Sensitive. Es macht einen Unterschied, ob eine Variable Player oder player heißt. Generell beeinflusst die Benennung die Funktion des Programms nicht, allerdings ist der Code für andere einfacher zu lesen, wenn man sich an die üblichen Standards hält:

  1. Methoden und Klassen: Pascal Case zum Beispiel „GetDetails“ oder „GetBonus“

  2. Variablen und Argumente in Methoden: Camel Case zum Beispiel „firstName“ oder „userId“


Funktionen in C#

Funktionen sind kleine Teile eines Programms, die eine spezifische Aufgabe erledigen. Sie sind immer aufrufbar und können zB eine Aufgabe auf die Konsole tätigen. Funktionen sind werden so aufgebaut:

public static void meineFunktion() {

}

Variablen in C#

Variablen sind Speicherbereiche, in denen Daten gespeichert und wieder ausgelesen werden. Innerhalb von Programmen können Variablen ständig ihre zugewiesenen Werte ändern.
Die Syntax für das Deklarieren von Variablen sieht folgendermaßen aus:
<Datentyp> <Variablenname> = <Initialisierungswert>;
In C# gibt es mehrere Variablentypen. Diese werden später noch genauer erklärt, zur Übersicht gibt es hier eine kurze Liste der wichtigsten Variablentypen:

Datentyp Erklärung

int

Ganze Zahlen

double

Kommazahlen

boolean

Kann nur die zwei Zustände true oder false annehmen

char

ein einzelnes Zeichen

string

Eine Folge von chars/Zeichen, also Wörter oder Sätze
